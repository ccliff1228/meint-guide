# ERP와 MES 개요 — 퀴즈 문제 예시

>
> 아래는 섹션별 대표 문제 1개씩, 총 6문제를 선별한 예시입니다.

---

## 1. 핵심 구분: 데이터 소스 (Q1~Q6)

**Q1.** ERP와 MES를 구분하는 가장 본질적인 기준은 무엇이며, 이 기준이 두 시스템의 역할 차이를 어떻게 결정짓는지 설명하시오.

> **모범 답안:** 데이터 소스(Data Source)이다. ERP는 사람이 입력한 데이터를 다루고, MES는 설비가 보내는 신호를 다룬다. 이 차이로 인해 ERP는 일/주 단위 계획·실적 집계, MES는 초/분 단위 실시간 모니터링·제어 역할을 수행한다.

---

## 2. ERP 공정관리 — 수동 입력 데이터 기반 (Q7~Q14)

**Q11.** ERP의 Quality Inspection 기능이 MES의 공정 인터록(Interlock) 기능과 어떻게 다른지, 그리고 왜 다른지 데이터 소스 관점에서 설명하시오.

> **모범 답안:** ERP의 Quality Inspection은 작업자가 검사 결과를 수동 입력하는 기록 관리 기능이다. MES의 공정 인터록은 선행 공정 미완료나 검사 불합격 시 후속 공정을 자동 차단하는 실시간 제어 기능이다. ERP는 사람 입력 데이터로 사후 관리하고, MES는 설비 신호로 실시간 제어하기 때문이다.

---

## 3. MES 공정관리 — 설비 데이터 기반 (Q15~Q21)

**Q15.** OEE(Overall Equipment Effectiveness)의 세 가지 구성 요소를 설명하고, MES가 이를 자동 산출할 수 있는 이유를 서술하시오.

> **모범 답안:** 가동률(Availability), 성능률(Performance), 양품률(Quality)이다. MES는 설비 신호(ON/OFF, 사이클 완료 펄스, 에러 코드 등)를 자동 수집하므로 가동 시간, 실제 생산량, 불량 수를 실시간 파악하여 OEE를 자동 계산할 수 있다.

---

## 4. 도입 판단 (Q22~Q30)

**Q22.** "오늘 Work Order 완료·진행·불량 건수" 대시보드는 ERP만으로 충분하고, "현재 3번 설비 가동/대기 상태" 대시보드에는 MES가 필요한 이유를 비교 설명하시오.

> **모범 답안:** Work Order 완료·진행·불량 건수는 작업자가 ERP에 입력한 집계성 데이터로 산출 가능하므로 ERP만으로 충분하다. 반면 현재 설비 가동/대기 상태는 설비가 보내는 실시간 신호를 수집해야만 파악 가능하므로 MES가 필요하다.

---

## 5. 분리 운영의 구조적 문제 — AS-IS (Q31~Q39)

**Q37.** 데이터 단절의 네 가지 문제(재고 불일치, 마스터 데이터 단절, 가시성 분산, 통합 장벽) 사이의 인과 관계를 설명하시오.

> **모범 답안:** 통합 장벽(비용·기간·인력 부족)이 근본 원인이다. 통합이 이루어지지 않으면 마스터 데이터 단절이 발생하고, 마스터 데이터가 동기화되지 않으면 재고 불일치가 생기며, 양쪽 데이터가 분산되어 가시성 분산 문제가 발생한다.

---

## 6. 통합 해결 방향 — TO-BE (Q40~Q50)

**Q49.** AX에서 "도메인 전문가가 AI에게 직접 지시"하는 것이 DX의 "도메인 전문가-개발자 간 번역 필요"와 비교하여 어떤 구조적 변화를 가져오는지 설명하시오.

> **모범 답안:** DX에서는 도메인 전문가 → 요구사항 문서 → 개발자 → 구현의 간접 경로를 거치며 번역 비용이 발생했다. AX에서는 도메인 전문가가 AI에 직접 지시하여 구현하므로 중간 번역 단계가 제거된다. 구현 주체가 '전문 개발팀'에서 '도메인 전문가 + AI'로 전환되어 소통 비용이 근본적으로 해소된다.
